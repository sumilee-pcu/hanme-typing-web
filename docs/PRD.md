# PRD: 레트로 한글 타자연습 웹앱

## 1. 제품 개요

### 제품명

가칭: 한메 타자 웹

최종 공개명은 기존 상표/프로그램명과 혼동되지 않도록 별도 검토한다.

### 한 줄 설명

1990년대 DOS/초기 Windows 한글 타자연습의 감성을 현대 웹에서 다시 경험할 수 있는 클린룸 방식의 한글 타자연습 앱.

### 배경

과거의 한메타자교사/한글타자/한컴타자연습 계열 프로그램은 자리연습, 낱말연습, 긴글연습, 게임형 연습을 통해 한글 타자 학습 경험을 대중화했다. 현재 사용자는 설치 없이 브라우저에서 바로 실행되는 가벼운 타자연습 도구를 기대한다. 이 제품은 원본 실행파일, 이미지, 문장 데이터, 사운드, 코드, 상표를 복제하지 않고, 기능적 아이디어와 레트로 감성만 새롭게 구현한다.

## 2. 목표

### 제품 목표

- 브라우저에서 바로 실행되는 한글 타자연습 MVP를 제공한다.
- 두벌식 한글 입력 기준으로 자리연습, 낱말연습, 짧은글/긴글연습, 게임형 연습을 지원한다.
- 레트로한 UI 분위기를 제공하되 원본 그래픽과 문구를 복제하지 않는다.
- 타수, 정확도, 오타, 진행률, 최고기록을 직관적으로 보여준다.
- 데스크톱 키보드 사용 경험을 우선으로 하되 모바일에서도 조회와 일부 연습이 가능하게 한다.

### 비목표

- 원본 DOS 프로그램을 웹 에뮬레이터로 배포하지 않는다.
- 원본 프로그램의 이름, 로고, 그래픽, 문장, 사운드, 실행파일을 포함하지 않는다.
- 회원가입, 서버 저장, 랭킹 시스템은 MVP 범위에서 제외한다.
- 세벌식, 외국어 타자, AI 문장 생성은 초기 버전 범위에서 제외한다.

## 3. 대상 사용자

### 1차 사용자

- 30~50대 사용자: 과거 한글 타자연습 프로그램을 기억하고, 추억과 실용을 함께 원하는 사용자.
- 학생/초보자: 한글 키보드 자리와 기본 타자 속도를 익히려는 사용자.
- 교사/강사: 수업 중 설치 없이 사용할 수 있는 간단한 타자연습 도구가 필요한 사용자.

### 사용자 니즈

- 설치 없이 바로 연습하고 싶다.
- 한글 자판 위치를 눈으로 보며 익히고 싶다.
- 연습 결과를 즉시 확인하고 싶다.
- 옛 프로그램 같은 분위기는 원하지만, 현재 브라우저에서 안정적으로 동작하기를 원한다.

## 4. 핵심 사용자 흐름

### 첫 진입

1. 사용자가 웹앱에 접속한다.
2. 첫 화면에서 연습 모드를 선택한다.
3. 사용자는 자리연습, 낱말연습, 긴글연습, 게임 중 하나를 시작한다.
4. 연습 종료 후 결과 화면에서 타수, 정확도, 오타 수, 소요 시간을 확인한다.
5. 사용자는 다시하기 또는 다른 모드로 이동한다.

### 자리연습

1. 사용자가 자음/모음/기본자리/전체자리 중 하나를 선택한다.
2. 화면에 목표 키와 가상 키보드가 표시된다.
3. 올바른 키를 누르면 다음 글쇠로 진행된다.
4. 일정 횟수 완료 후 결과를 표시한다.

### 낱말연습

1. 사용자가 난이도를 선택한다.
2. 단어가 하나씩 제시된다.
3. 사용자는 입력창에 단어를 입력한다.
4. Enter 또는 자동완성 조건으로 다음 단어로 넘어간다.
5. 제한 시간 또는 목표 단어 수를 완료하면 결과를 표시한다.

### 긴글연습

1. 사용자가 연습 지문을 선택한다.
2. 원문과 입력 영역이 표시된다.
3. 현재 입력 위치, 맞은 글자, 틀린 글자를 실시간으로 표시한다.
4. 지문 완료 후 평균 타수와 정확도를 표시한다.

### 게임형 연습

1. 사용자가 게임 모드를 선택한다.
2. 화면 위에서 단어가 내려오거나 등장한다.
3. 사용자가 해당 단어를 입력하면 단어가 제거되고 점수가 증가한다.
4. 생명/시간/난이도 조건에 따라 게임이 종료된다.

## 5. 기능 요구사항

### FR-1. 홈/모드 선택

- 사용자는 홈 화면에서 연습 모드를 선택할 수 있어야 한다.
- 각 모드는 아이콘 또는 시각적 구분을 가진다.
- 최근 최고기록이 있는 경우 홈에서 간단히 표시한다.

### FR-2. 자리연습

- 두벌식 한글 키보드 배열을 화면에 표시한다.
- 현재 눌러야 할 키를 강조한다.
- 정타/오타를 즉시 계산한다.
- 기본 단계는 다음을 포함한다.
  - 기본자리
  - 왼손 자리
  - 오른손 자리
  - 모음
  - 자음
  - 전체자리

### FR-3. 낱말연습

- 난이도별 단어 목록을 제공한다.
- 초급 단어는 1~2음절 중심, 중급은 3~4음절, 고급은 겹받침/복합모음 포함 단어로 구성한다.
- 원본 프로그램의 단어 목록을 사용하지 않고 새 단어 목록을 작성한다.
- 입력 중 오타 여부를 실시간 표시한다.

### FR-4. 짧은글/긴글연습

- 저작권 문제가 없는 자체 제작 지문 또는 퍼블릭 도메인 지문만 사용한다.
- 사용자는 지문 목록에서 선택할 수 있다.
- 입력 진행률, 현재 줄, 남은 글자 수를 표시한다.
- 줄바꿈과 공백을 연습 평가에 포함할지 설정할 수 있다.

### FR-5. 게임형 연습

- MVP에서는 "단어 방어" 게임 1종을 제공한다.
- 단어가 화면 상단에서 하단으로 이동한다.
- 사용자가 단어를 정확히 입력하면 해당 단어가 사라지고 점수가 올라간다.
- 단어가 하단에 도달하면 생명이 감소한다.
- 난이도에 따라 낙하 속도와 동시 등장 단어 수가 증가한다.

### FR-6. 결과/통계

- 연습 종료 후 다음 값을 표시한다.
  - 평균 타수
  - 정확도
  - 정타 수
  - 오타 수
  - 소요 시간
  - 점수
- 브라우저 로컬 저장소에 모드별 최고기록을 저장한다.
- 사용자는 기록을 초기화할 수 있다.

### FR-7. 설정

- 사용자는 사운드 켜기/끄기를 선택할 수 있다.
- 사용자는 테마를 선택할 수 있다.
  - 레트로 DOS
  - 밝은 화면
  - 고대비
- 사용자는 제한 시간 또는 목표 글자 수를 선택할 수 있다.

## 6. 비기능 요구사항

### 성능

- 첫 로딩은 일반적인 데스크톱 브라우저 기준 2초 이내를 목표로 한다.
- 게임 모드는 60fps에 가깝게 동작해야 한다.
- 네트워크 없이도 기본 연습 데이터가 동작해야 한다.

### 호환성

- 최신 Chrome, Edge, Safari, Firefox를 지원한다.
- 데스크톱 키보드 환경을 1차 지원한다.
- 모바일은 홈, 결과 확인, 간단 연습을 지원하되 게임 모드는 데스크톱 최적화로 둔다.

### 접근성

- 키보드만으로 주요 화면을 조작할 수 있어야 한다.
- 색상만으로 정오답을 구분하지 않고 텍스트/형태 변화를 함께 사용한다.
- 고대비 테마를 제공한다.

### 개인정보

- MVP에서는 회원가입과 서버 저장을 제공하지 않는다.
- 모든 기록은 브라우저 로컬 저장소에만 저장한다.

## 7. 저작권/상표 원칙

### 클린룸 원칙

- 원본 실행파일, 바이너리, 리소스, 이미지, 폰트, 사운드, 문장 데이터, 단어 목록을 사용하지 않는다.
- 원본 화면을 픽셀 단위로 복제하지 않는다.
- "한메타자", "한컴타자", "한글과컴퓨터" 등 기존 상표 또는 서비스명과 혼동될 수 있는 명칭을 공개 제품명으로 사용하지 않는다.
- 기능적 아이디어는 일반적인 타자연습 도구의 범위에서 새롭게 구현한다.
- 개발 문서에는 원본 참고 이미지를 링크 수준으로만 남기고 저장소에 포함하지 않는다.

### 콘텐츠 원칙

- 단어 목록은 새로 작성한다.
- 긴글 지문은 자체 작성, 공공누리/CC0/퍼블릭 도메인 등 명확한 라이선스의 텍스트만 사용한다.
- 레트로 효과음은 직접 제작하거나 명확한 오픈 라이선스 리소스만 사용한다.

## 8. UX/UI 방향

### 시각 컨셉

- DOS/초기 GUI를 연상시키는 단순한 색면, 굵은 테두리, 픽셀 느낌의 타이포그래피를 사용한다.
- 단, 원본 프로그램의 화면 구성과 색상 조합을 그대로 복제하지 않는다.
- 첫 화면은 앱 기능으로 바로 들어가는 도구형 화면으로 구성한다.

### 주요 화면

- 홈: 모드 선택, 최고기록 요약, 설정 진입
- 자리연습: 큰 목표 글쇠, 가상 키보드, 정확도/타수 패널
- 낱말연습: 현재 단어, 입력창, 진행률, 통계
- 긴글연습: 원문 패널, 입력 영역, 진행률
- 게임: 전체 화면형 플레이 영역, 점수/생명/시간 표시
- 결과: 요약 점수, 최고기록 갱신 여부, 다시하기 버튼

## 11. 성공 지표

- 첫 방문자가 10초 이내에 첫 연습을 시작할 수 있다.
- 자리연습/낱말연습/긴글연습/게임을 각각 오류 없이 완료할 수 있다.
- 연습 결과가 실제 입력값과 일관되게 계산된다.
- 브라우저 새로고침 후 최고기록이 유지된다.
- 원본 프로그램의 저작물 요소가 저장소에 포함되지 않는다.

## 12. 기술 방향

### 권장 스택

- Vite
- TypeScript
- React
- Canvas API 또는 DOM 기반 게임 루프
- LocalStorage 또는 IndexedDB
- Vitest
- Playwright

### 구현 메모

- 한글 조합 입력은 `compositionstart`, `compositionupdate`, `compositionend`, `beforeinput`, `input`, `keydown` 이벤트를 함께 검증해야 한다.
- 타수 계산은 완성형 글자 기준과 실제 키 입력 기준 중 하나를 명확히 선택한다. MVP에서는 완성형 글자 기준 CPM을 기본으로 하고, 추후 KPM 옵션을 검토한다.
- 게임 루프는 `requestAnimationFrame`을 사용한다.
- 입력 판정 로직은 UI와 분리해 단위 테스트가 가능하게 한다.

## 14. 주요 리스크

### 저작권/상표 리스크

- 원본과 지나치게 유사한 이름, 화면, 문장, 사운드는 분쟁 가능성이 있다.
- 대응: 클린룸 원칙을 문서화하고, 모든 리소스 출처를 기록한다.

### 한글 입력 판정 리스크

- 브라우저와 OS별 한글 IME 동작이 다를 수 있다.
- 대응: Chrome/Safari/Firefox에서 조합 이벤트 테스트를 수행한다.

### 레트로 UI와 사용성 충돌

- 감성은 좋지만 실제 사용성이 떨어질 수 있다.
- 대응: 레트로 테마는 유지하되 버튼, 입력, 결과는 현대 웹 기준의 명확한 피드백을 제공한다.

## 15. MVP 완료 기준

- 사용자는 브라우저에서 앱을 열고 네 가지 모드 중 하나를 선택할 수 있다.
- 각 모드는 시작, 진행, 종료, 결과 확인 흐름을 가진다.
- 결과 값은 테스트 가능한 로직으로 계산된다.
- 새로고침 후에도 최고기록이 남아 있다.
- 저장소에는 원본 프로그램의 실행파일/이미지/사운드/문장 데이터가 없다.
- README에 저작권 클린룸 원칙과 리소스 라이선스 정책이 명시되어 있다.
